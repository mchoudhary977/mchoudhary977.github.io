  1. Normalization

Set of rules that cover some of the most fundamental structural issues with RDMS designs.

To be in a stricter normal form, you need to also conform to the lesser form, though none of these rules are ever followed one hundred percent of the time.

Two families of normalization concepts:-

  1. Rules Covering the shape of a table

First Normal Form:-

  • All columns must be atomic
  • All rows of a table must contain the same number of values - no arrays

  1. Rules covering the relationship of non-key attributes to key attributes

Second Normal Form:-

  • All attributes must be a fact about the entire primary key and not a subset of the primary key

Third Normal Form:-

  • All attributes must be a fact about the entire primary key and not any non-primary key attributes.

Boyce-Codd Normal Form:-

  • Every candidate key is identified, all attributes are fully dependent on a key and all columns must identify a fact about a key and nothing but a key.

Third Normal is referred to as the most important normal form, each table has one meaning, and each scalar attribute refers to the entire natural key of the final objects.

  1. Schema

Schemas allow to group together objects for security and logical ordering. By default, there is a schema in every database - dbo.

GO - GO is not a statement in T-SQL, it is batch separator that splits your queries into multiple server communications, so it does not need termination.

  1. NULL

NULL is special value that mathematically means UNKNOWN.

  • NULL = NULL : will return FALSE as Unknown value cannot be compared.
  • NULL values not allowed in PRIMARY KEY constraint.
  • To allow NULL values for PRIMARY KEY column, set below DB property ON:
  • SET ANSI_NULL_DFLT_ON

  1. DataTypes
  1. Precise Numeric [Stores number-based data with loss of precision]
  • bit - 1, 0 or NULL
  • tinyint : 0 and 255 [1 byte]
  • smallint : -32,768 and 32,767 [2 byte]
  • int : -2,147,483,648 and 2,147,483,647 [4 bytes]
  • bigint : -9,223,372,036,854,775,808 and 9,223,372,036,854,775,807 [8 bytes]
  • decimal : -10^38-1 and 10^38-1 : decimal(38,37) - max precission
  • money : -922,337,203,685,477.5808 to 922,337,203,685,477.5807 [8 bytes]
  • smallmoney : -214,748.3648 to 214,748.3674 [4 bytes]

  1. Approximate numeric data [Stores approximation of numbers]
  • float(N)
  • real

  1. Date and time values
  • date : January 1, 0001 to December 31, 9999 [3 bytes]
  • time(N) : time(7) - HH:MM:SS.0000001 [3 to 5 bytes]
  • datetime2(N) : accuracy just like time above
  • datetimeoffset : includes an offset for time zone
  • smalldatetime : January 1, 1900 to June 6, 2079 - accuracy to 1 minute
  • datetime : January 1, 1753 to December 31, 9999 - accuracy to 3,33 milliseconds

  1. Binary data
  • binary(N) : fixed length binary data , max value of N allowed - 8000 i.e. 8000 byte long binary value
  • varbinary(N): Variable length with max allowed value of N - 8000
  • varbinary(max): Variable length binary data upto (2^31) -1 bytes (2GB) long.

  1. Character (or string) data
  • char(N)
  • varchar(N)
  • varchar(max)
  • nchar, nvarchar, nvarchar(max) : unicode equivalents of char, varchar and varchar(max), has more than 256 characters of ASCII

  1. Other DataTypes:
  • sql_variant : store nearly any data type. Max Length - 8016 bytes
  • rowversion (timestamp is a synonym) : value in rowversion data type based column changes on every modification of the row
  • uniqueidentifier : stores a globally unique identifier (GUID) value.
  • XML : store xml document in a column value
  • Spatial type : geometry, geographical data like shapes, maps, lines
  • hierarchyId : store data about a hierarchy, along with providing methods for manipulating the hierarchy

  1. Computed Columns
  • Columns based on expressions
  • Computed columns let you manifest an expression as a column for usage

CREATE TABLE EXAMPLES.TABLE1

(

        FIRSTNAME nvarchar(50) NULL,

        LASTNAME nvarchar(50) NULL,

        FULLNAME AS CONCAT(LASTNAME,’,’+FIRSTNAME)

);

ALTER TABLE EXAMPLES.TABLE1 DROP COLUMN FULLNAME;

ALTER TABLE EXAMPLES.TABLE1

        ADD FULLNAME AS CONCAT(LASTNAME,’,’+FIRSTNAME) PERSISTED;

  • To save evaluated expression in physical storage along with table data, add PERSISTED.

  1. Dynamic Data Masking
  • Lets you mask data in a column from the view of the user.
  • Who can see the data masked or unmasked is controlled by a database level permission called UNMASK.
  • Four Types of data mask functions
  • Default
  • Takes default mask of datatype

ALTER TABLE EXAMPLES.DATAMASKING ALTER COLUMN FIRSTNAME

        ADD MASKED WITH (FUNCTION=’default()’);

  • Email

ALTER TABLE EXAMPLES.DATAMASKING ALTER COLUMN EMAILADDRESS

        ADD MASKED WITH (FUNCTION=’email()’);

  • Random

ALTER TABLE EXAMPLES.DATAMASKING ALTER COLUMN CARCOUNT

        ADD MASKED WITH (FUNCTION=’random(1,3)’);

  • Partial

ALTER TABLE EXAMPLES.DATAMASKING ALTER COLUMN LASTNAME

        ADD MASKED WITH (FUNCTION=’partial(3,”____”,2)’);

–To mask entire value

ALTER TABLE EXAMPLES.DATAMASKING ALTER COLUMN EMAILADDRESS

        ADD MASKED WITH (FUNCTION=’partial(0,”Unknown”,0)’);
